# Команда Migrations - гра "Що? Де? Коли?"

## Склад команди

- Анна - капітан команди.
- Тимофій.
- Поліна.
- Максим.

## Правила гри

Це командна гра по Django models, міграціях і admin.

- Капітан визначає відповідального за кожне питання.
- Відповідальний озвучує фінальну відповідь команди.
- Обговорювати відповідь може вся команда.
- Не можна, щоб на всі питання відповідав тільки капітан або найшвидший учасник.
- Відповідь має бути короткою: 1-3 речення.

Система балів:

- правильна відповідь - 1 бал;
- якщо команда не відповіла, інша команда може перехопити питання за 0.5 бала.

### Питання 2

### Питання 3

Чому Django не змінює базу даних автоматично одразу після кожної зміни у `models.py`?

### Питання 4

Є модель `Delivery`:

```python
class Delivery(models.Model):
    address = models.CharField(max_length=255)
    price = models.DecimalField(max_digits=8, decimal_places=2)
```

Після того як у базі вже є записи, розробник додає поле:

```python
status = models.CharField(max_length=20, default="new")
```

Навіщо тут `default="new"` і які дві команди треба виконати після зміни моделі?

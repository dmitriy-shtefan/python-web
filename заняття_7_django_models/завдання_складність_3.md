# Завдання складність 3

## Мета

Створити повніший data baseline проєкту без зв'язків між models.

До кінця роботи має бути:

- 3 окремі models;
- 5-8 полів у кожній model;
- різні типи полів;
- коректний `__str__` у кожній model;
- щонайменше 3 файли міграцій;
- усі models зареєстровані в admin;
- 15-20 реалістичних записів загалом;
- усне пояснення структури даних.

## Завдання

1. Оберіть 3 сутності для вашої предметної області.

   Вони мають бути логічно пов'язані темою проєкту.

2. Для кожної сутності створіть model.

   Вимоги до кожної model:

   - назва class у `PascalCase`;
   - поля з осмисленими назвами;
   - не менше 5 полів;
   - не всі поля текстові;
   - є `blank=True` там, де поле може бути порожнім;
   - є `default` там, де є логічне стартове значення;
   - є `__str__`.

3. Зробіть першу міграцію і застосуйте її.

4. Зареєструйте всі 3 models в admin.

5. Створіть мінімум 5 записів для кожної model.

   Заборонені тестові значення:

   - `test`;
   - `aaa`;
   - `123`;
   - порожні записи без сенсу.

6. Після наповнення admin змініть кожну model один раз:

   - в одну model додайте нове поле;
   - в іншій model додайте `default` до наявного поля;
   - у третій model додайте або змініть `blank=True`.

7. Зробіть ще одну міграцію і застосуйте її.

8. Перевірте в admin, що:

   - усі models відкриваються;
   - записи читаються нормально;
   - `__str__` працює;
   - нові або змінені поля видно в admin;
   - старі записи не зламалися після другої міграції.

9. Якщо залишився час, зробіть третю зміну model і третю міграцію.

   Варіанти:

   - додати поле дати;
   - додати boolean-поле;
   - перейменувати невдале поле;
   - змінити `max_length`;
   - змінити логіку `__str__`.

## Що здати

Показати викладачу:

- `models.py`;
- папку `migrations`;
- admin зі списком усіх 3 models;
- по одному відкритому запису з кожної model;
- приклад запису, який був створений до другої міграції і залишився робочим після неї.

Усно пояснити:

- чому обрані саме ці 3 models;
- чому поки немає зв'язків між ними;
- які поля є обов'язковими, а які можуть бути порожніми;
- що змінила кожна міграція;
- які помилки в структурі даних ви помітили під час наповнення admin.

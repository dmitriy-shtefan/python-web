# Завдання складність 2

## Мета

Покращити одну основну model так, щоб вона була схожа на реальну сутність предметної області.

До кінця роботи має бути:

- одна model з 5-7 осмисленими полями;
- щонайменше 3 різні типи полів;
- коректний `__str__`;
- дві послідовні міграції;
- 5-7 якісних записів в admin;
- коротке пояснення вибору полів.

## Завдання

1. Оберіть одну головну сутність проєкту.

2. Переробіть model так, щоб у ній були не тільки текстові поля.

Можна використовувати:

- `CharField`;
- `TextField`;
- `IntegerField`;
- `DecimalField`;
- `BooleanField`;
- `DateField`;
- `DateTimeField`.

Орієнтир:

```python
from django.db import models


class Product(models.Model):
    title = models.CharField(max_length=200)
    description = models.TextField(blank=True)
    price = models.DecimalField(max_digits=8, decimal_places=2)
    quantity = models.IntegerField(default=0)
    is_available = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title
```

3. Виконайте першу міграцію:

```bash
python manage.py makemigrations
python manage.py migrate
```

4. Зареєструйте model в admin, якщо вона ще не зареєстрована:

```python
from django.contrib import admin
from .models import Product


admin.site.register(Product)
```

5. Створіть 5-7 реалістичних записів через admin.

6. Змініть model ще раз: додайте одне просте поле.

Наприклад:

```python
rating = models.IntegerField(default=0)
```

або:

```python
notes = models.TextField(blank=True)
```

7. Виконайте другу міграцію:

```bash
python manage.py makemigrations
python manage.py migrate
```

8. Перевірте, що нове поле видно в admin і його можна заповнити.

## Що здати

Показати викладачу:

- `models.py`;
- два файли міграцій;
- admin зі списком записів;
- один відкритий запис з усіма полями.

Усно пояснити:

- яку сутність описує model;
- чому обрані саме такі типи полів;
- що змінилося між першою і другою міграцією;
- навіщо потрібен `__str__`.

# Інструкція для студента: від 0 до мінімального baseline

Цей файл для ситуації, коли твій персональний проєкт ще не готовий або майже не запускається.
Мета - довести проєкт до мінімального робочого стану.

В результаті має бути:

- Django-проєкт запускається;
- є хоча б один app;
- є головна сторінка;
- є templates;
- є static file;
- є одна model;
- виконані `makemigrations` і `migrate`;
- model зареєстрована в admin;
- в admin створено 2-3 тестові записи.

## 0. Якщо проєкт уже існує

Якщо в тебе вже є Django-проєкт, не створюй новий одразу.

Спочатку перевір:

```bash
python manage.py runserver
```

Якщо команда працює і сайт відкривається - переходь до розділу `3. Перевір app`.

Якщо команда не працює:

- переконайся, що ти в папці, де є `manage.py`;
- прочитай перший рядок помилки;
- якщо Django не встановлений, активуй своє віртуальне середовище і виконай `python -m pip install django`;
- якщо `python` не працює, спробуй `python3`.

## 1. Створи новий мінімальний проєкт

Використовуй короткі англійські назви без пробілів.

Приклад:

- назва папки: `shop`, `store`
- назва app: `products`, `orders`

Команди для macOS / Linux:

```bash
mkdir shop
cd shop
python -m venv .venv
source .venv/bin/activate
python -m pip install django
django-admin startproject shop .
python manage.py startapp products
```

Якщо `python` не працює:

```bash
python3 -m venv .venv
source .venv/bin/activate
python3 -m pip install django
django-admin startproject shop .
python3 manage.py startapp products
```

Команди для Windows PowerShell:

```powershell
mkdir shop
cd shop
python -m venv .venv
.venv\Scripts\Activate.ps1
python -m pip install django
django-admin startproject shop .
python manage.py startapp products
```

Перевір запуск:

```bash
python manage.py runserver
```

Мінімальний успіх: у браузері відкривається стандартна стартова сторінка Django.

## 2. Додай app у `INSTALLED_APPS`

Відкрий файл:

```text
shop/settings.py
```

Знайди `INSTALLED_APPS` і додай туди `products`:

```python
INSTALLED_APPS = [
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",
    "products",
]
```

Мінімальний успіх: після цього `python manage.py runserver` не падає з помилкою.

## 3. Перевір app

У папці app мають бути файли:

```text
products/
    admin.py
    apps.py
    models.py
    views.py
```

Якщо app називається не `products`, використовуй свою назву app далі в інструкції.

## 4. Зроби головну сторінку

У файлі `products/views.py` напиши:

```python
from django.shortcuts import render


def home(request):
    return render(request, "products/home.html")
```

Створи файл `products/urls.py`:

```python
from django.urls import path
from . import views


urlpatterns = [
    path("", views.home, name="home"),
]
```

У файлі `shop/urls.py` підключи app:

```python
from django.contrib import admin
from django.urls import include, path


urlpatterns = [
    path("admin/", admin.site.urls),
    path("", include("products.urls")),
]
```

Мінімальний успіх: Django знає, який view відкривати на головній сторінці.

## 5. Додай template

Створи папки:

```text
products/
    templates/
        products/
            home.html
```

У файлі `products/templates/products/home.html` напиши:

```html
{% load static %}

<!DOCTYPE html>
<html lang="uk">
<head>
    <meta charset="UTF-8">
    <title>Мій персональний проєкт</title>
    <link rel="stylesheet" href="{% static 'products/style.css' %}">
</head>
<body>
    <h1>Мій персональний проєкт</h1>
    <p>Це мінімальна головна сторінка проєкту.</p>
</body>
</html>
```

Перевір:

```bash
python manage.py runserver
```

Мінімальний успіх: замість стандартної сторінки Django відкривається твоя HTML-сторінка.

## 6. Додай static file

Створи папки:

```text
products/
    static/
        products/
            style.css
```

У файлі `products/static/products/style.css` напиши:

```css
body {
    font-family: Arial, sans-serif;
    max-width: 800px;
    margin: 40px auto;
    line-height: 1.5;
}

h1 {
    color: #234;
}
```

Перезапусти сервер і онови сторінку.

Мінімальний успіх: текст на сторінці має інший стиль, ніж стандартний браузерний.

## 7. Створи першу model

Для мінімального baseline достатньо однієї моделі з 3-5 полями.

Якщо ще не знаєш тему проєкту, тимчасово створи модель `Item`.
Потім її можна буде замінити на більш конкретну: `Product`, `Task`, `Course`, `Book`, `Doctor`, `Event`.

У файлі `products/models.py` напиши:

```python
from django.db import models


class Item(models.Model):
    title = models.CharField(max_length=200)
    description = models.TextField(blank=True)
    is_active = models.BooleanField(default=True)
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return self.title
```

Мінімальний успіх: у тебе є Python-клас, який описує дані майбутньої таблиці.

## 8. Створи і застосуй міграції

Виконай:

```bash
python manage.py makemigrations
python manage.py migrate
```

Якщо `makemigrations` пише `No changes detected`, перевір:

- app додано в `INSTALLED_APPS`;
- модель збережена у правильному файлі `products/models.py`;
- у класі моделі немає синтаксичної помилки;
- сервер не обов'язково має бути запущений під час міграцій.

Мінімальний успіх: Django створив файл міграції і застосував її до бази даних.

## 9. Зареєструй model в admin

У файлі `products/admin.py` напиши:

```python
from django.contrib import admin
from .models import Item


admin.site.register(Item)
```

Створи admin-користувача:

```bash
python manage.py createsuperuser
```

Якщо просить email, можна ввести свій email або пропустити, якщо поле не обов'язкове.
Пароль під час введення може не відображатися - це нормально.

Запусти сервер:

```bash
python manage.py runserver
```

Відкрий:

```text
http://127.0.0.1:8000/admin/
```

Мінімальний успіх: після входу в admin ти бачиш модель `Item`.

## 10. Створи 2-3 записи

В admin відкрий `Items` і створи кілька записів.

Наприклад:

- `Перший запис`
- `Другий запис`
- `Тестовий запис`

Мінімальний успіх: записи зберігаються, відображаються списком і мають нормальні назви, а не `Item object (1)`.

## 11. Фінальна перевірка перед кінцем пари

Виконай цей чекліст:

- `python manage.py runserver` запускається без помилок;
- головна сторінка відкривається;
- template реально використовується;
- static CSS підключений;
- app є в `INSTALLED_APPS`;
- у `models.py` є одна модель;
- `makemigrations` виконано;
- `migrate` виконано;
- model зареєстрована в `admin.py`;
- superuser створений;
- в admin є 2-3 записи.

Якщо все це виконано, твій проєкт досяг мінімального baseline для наступних занять.

## 12. Якщо застряг

Спочатку визнач, на якому кроці ти застряг:

1. Не створюється проєкт.
2. Не запускається сервер.
3. Не відкривається головна сторінка.
4. Не працює template.
5. Не підключився static.
6. Не створюється міграція.
7. Не відкривається admin.
8. Model не видно в admin.

Потім покажи викладачу:

- команду, яку ти виконав;
- повний текст помилки;
- файл, який редагував перед помилкою.

Так проблему можна буде виправити набагато швидше.

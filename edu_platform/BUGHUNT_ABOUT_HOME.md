# Bughunt: `home`, `about`, `extends`, `include`, static files

## Мета

Полагодити сторінки `/courses/home/` та `/courses/about/`, щоб вони:

- відкривалися без template-помилок;
- використовували спільний `courses/base.html`;
- мали navigation через `{% include %}`;
- підключали CSS з `courses/static/courses/css/site.css`.

## Правила

- Не переписуйте все з нуля.
- Не чіпайте `views.py` і `urls.py`, якщо вони вже відкривають правильні сторінки.
- Виправляйте одну помилку за раз.
- Після кожного виправлення оновлюйте сторінку в браузері.
- Не додавайте inline CSS замість static-файлу.

## Порядок роботи

1. Запустіть сервер:

```bash
python manage.py runserver
```

2. Відкрийте:

```text
http://127.0.0.1:8000/courses/home/
```

3. Прочитайте першу помилку Django.
4. Знайдіть файл, який її спричинив.
5. Виправте тільки цю помилку.
6. Повторіть перевірку для:

```text
http://127.0.0.1:8000/courses/about/
```

## Що потрібно знайти

У коді є помилки в таких місцях:

- шлях у `{% extends %}`;
- позиція `{% extends %}` у дочірньому шаблоні;
- назва `{% block %}`;
- шлях у `{% include %}`;
- підключення static template tag library;
- шлях до CSS у `{% static %}`.

## Очікуваний результат

- `/courses/home/` відкривається.
- `/courses/about/` відкривається.
- Обидві сторінки мають спільний header, navigation і footer.
- CSS реально змінює вигляд сторінок.
- Одна зміна в `navbar.html` відображається на обох сторінках.

# Домашнє завдання

## Мета

Закріпити базові `magic methods` у Python, підготувати перехід від ООП-практики до Django і зібрати робочу заготовку персонального проєкту.

## Завдання

1. Виконати одне з двох практичних завдань по ООП
2. Створити власний Django-проєкт за темою персонального проєкту (опціонально).
3. Додати один застосунок і перевірити, що dev-сервер запускається без критичних помилок (опціонально).

### Завдання (Складність 1)

Створити клас `Book` з полями `title`, `author`, `pages`.

Потрібно реалізувати:

- `__str__()` - повертати читабельний опис книги.
- `__len__()` - повертати кількість сторінок.
- `__call__(pages_per_hour)` - повертати приблизний час читання книги в годинах.

У файлі з рішенням:

- Створити щонайменше 2 об'єкти.
- Показати результат `print(book)`, `len(book)` і виклику `book(30)`.
- Додати 2-3 короткі рядки виводу, які пояснюють, що саме зараз перевіряється.

### Завдання (Складність 2)

Створити клас `CourseModule` з полями `title`, `duration_hours`, `practice_tasks`.

Потрібно реалізувати:

- `__str__()` - короткий опис модуля для користувача.
- `__repr__()` - технічне представлення об'єкта для відладки.
- `__len__()` - кількість практичних завдань у модулі.
- `__add__(other)` - сумарну тривалість двох модулів.
- `__eq__(other)` - порівняння модулів за кількістю практичних завдань.
- `__lt__(other)` - порівняння модулів за тривалістю, щоб можна було використовувати `sorted()`.

У файлі з рішенням:

- Створити мінімум 3 об'єкти.
- Показати роботу `repr(...)`, `len(...)`, додавання двох об'єктів і порівнянь `==` та `<`.
- Відсортувати список модулів через `sorted(...)`.
- У `__add__()` коректно обробити випадок, коли передали не об'єкт цього класу, через `NotImplemented`.
